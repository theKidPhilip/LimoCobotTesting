1) Launch LiDAR only (no base)

First, find what your LiDAR launch file is called:

ros2 pkg prefix ydlidar_ros2_driver
ros2 pkg prefix limo_bringup

Now try one of these (depending on what exists on your robot):

Option A (most common): launch ydlidar directly
ros2 launch ydlidar_ros2_driver ydlidar_launch.py

If that file name doesn’t exist, list available launch files:

ros2 pkg executables ydlidar_ros2_driver
ros2 pkg prefix ydlidar_ros2_driver
ls $(ros2 pkg prefix ydlidar_ros2_driver)/share/ydlidar_ros2_driver/launch

Then run the one you see there.

Option B: use the exact params file you were already using
ros2 run ydlidar_ros2_driver ydlidar_ros2_driver_node --ros-args --params-file \
$(ros2 pkg prefix ydlidar_ros2_driver)/share/ydlidar_ros2_driver/params/TminiPro.yaml
2) In a NEW terminal: run your wall follower only
ros2 run blazerbot_wall_follower wall_follower_node

(If your executable name is different, run ros2 pkg executables blazerbot_wall_follower to see it.)

3) Confirm inputs exist
Check LiDAR is publishing
ros2 topic info /scan
ros2 topic echo /scan --once

You must see Publisher count: 1 and a message print.

4) Confirm wall follower is publishing /cmd_vel

In another terminal:

ros2 topic info /cmd_vel
ros2 topic echo /cmd_vel

Expected:

Publisher count: 1

echo shows linear.x and angular.z changing over time

If you want a clean “one-shot” check:

ros2 topic echo /cmd_vel --once
5) If /cmd_vel is still NOT publishing

Two super common causes:

A) Your node is running but never enabling motion

Your current code has self.enabled = False and uses keyboard thread toggling. If you want no button press, you must force enable:

Add in __init__:

self.enabled = True
self.get_logger().info("Auto-enabled wall follower (no keypress).")

And in control_callback() add at the very top:

if not self.enabled:
    self.twist_cmd.linear.x = 0.0
    self.twist_cmd.angular.z = 0.0
    self.publish_twist_cmd()
    return
B) The LiDAR topic name isn’t /scan

Check:

ros2 topic list | grep -i scan

If it’s something like /ydlidar/scan, then change your subscriber topic accordingly.
